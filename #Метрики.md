# Полное руководство по метрикам когнитивно-моторного тренажёра

## 📋 Общая структура данных

### 1. CSV файл (сырые данные)
Назначение Детальная запись каждой попытки каждого теста
Формат UTF-8-BOM (для корректного открытия в Excel)
Структура Каждая строка - отдельная попытка

### 2. PDF отчёт (аналитика)
Назначение Аналитическая сводка с интерпретацией результатов
Формат Структурированный PDF с таблицами и выводами

## 📊 МЕТРИКИ CSV ФАЙЛА (сырые данные)

### Общие поля для всех тестов

 Поле  Тип  Описание  Как использовать 
---------------------------------------
 `test_name`  string  Название теста  Группировка данных по тестам 
 
 `attempt_number`  int  Номер попытки (1, 2, 3...)  Анализ прогресса обучения 
 
 `total_attempts`  int  Всего попыток в тесте  Нормализация показателей 
 
 `timestamp`  string  Метка времени (формат 000001, 000002...)  Временной анализ, порядок событий 
 
 `latency`  float (сек)  Время принятия решения - от появления стимула до начала движения  Ключевая метрика когнитивной обработки 
 
 `motor_time`  float (сек)  Моторное время - от начала движения до клика  Оценка моторных навыков 
 
 `total_rt`  float (сек)  Общее время реакции - сумма latency + motor_time  Общая скорость реакции 
 
 `correct`  boolean  Правильность ответа (TrueFalse)  Расчет точности 
 
 `anticipation`  boolean  Преждевременная реакция (до стимула)  Ошибки торможения 
 
 `delay`  boolean  Запаздывающая реакция (5 сек)  Проблемы внимания 
 
 `click_x`, `click_y`  float  Координаты клика (пиксели)  Пространственная точность

## 🧪 СПЕЦИФИЧЕСКИЕ МЕТРИКИ ПО ТЕСТАМ

### 1. Простая реакция (Simple Reaction)
Быстро отреагировать на появление стимула

Ключевые метрики

- `latency` - чистое время обработки информации
- 
- `motor_time` - чистое время двигательной реакции
- 
- Стабильность `latency` между попытками

Интерпретация
- Latency  0.25с - отличная скорость обработки
- Latency 0.25-0.35с - хорошая скорость
- Latency 0.35-0.45с - средняя скорость
- Latency  0.45с - требуется тренировка

### 2. Реакция выбора (Choice Reaction)
Суть теста Выбрать правильный стимул среди нескольких
Дополнительные метрики
- `choice_time = latency` - время принятия решения при выборе
- Увеличение `latency` относительно простой реакции = стоимость выбора

Интерпретация
- Choice time  0.4с - быстрый анализ вариантов
- Choice time 0.4-0.6с - хорошая скорость выбора
- Choice time 0.6-0.8с - средняя скорость
- Choice time  0.8с - трудности с принятием решений

### 3. Сложный выбор (Complex Choice)
Суть теста Выбор с отвлекающими факторами
Ключевые метрики
- `complex_time = latency` - время принятия сложного решения
- `accuracy` под нагрузкой - устойчивость внимания

Интерпретация
- Complex time  0.6с - отличная помехоустойчивость
- Complex time 0.6-0.8с - хорошая устойчивость
- Complex time 0.8-1.0с - средняя устойчивость
- Complex time  1.0с - трудности с фильтрацией помех

### 4. Реакция на движущийся объект (Moving Object)
Суть теста Попасть в движущуюся цель
Специфические метрики
 Поле  Описание  Интерпретация 
-------------------------------
 `timing_delay`  Временная ошибка (сек)  + - опоздание, - - упреждение 
 `distance_from_center`  Пространственная ошибка (px)  30px - отлично, 60px - хорошо 
 `trajectory_type`  Тип траектории объекта  Анализ предсказания движения 
 `speed`  Скорость объекта  Сложность условия 

Расчетные метрики в PDF
- `avg_timing_error` - средняя временная ошибка
- `timing_consistency` - стабильность времени реакции
- `avg_spatial_error` - средняя пространственная ошибка
- `spatial_precision` - пространственная точность (%)

### 5. Предвидение траектории (Trajectory Prediction)
Суть теста Предсказать конечную точку движущегося объекта
Специфические метрики
 Поле  Описание  Интерпретация 
-------------------------------
 `prediction_accuracy`  Точность предсказания (%)  80% - отлично 
 `prediction_error_px`  Ошибка в пикселях  20px - высокая точность 
 `observation_time`  Время наблюдения траектории  Анализ стратегии 

Расчетные метрики в PDF
- `extrapolation_skill` - навык экстраполяции (0-100%)
- `prediction_consistency` - стабильность предсказаний

### 6. Слежение (Tracking)
Суть теста Удерживать курсор на движущейся цели
Специфические метрики
 Поле  Описание  Интерпретация 
-------------------------------
 `avg_distance_px`  Среднее расстояние от цели  Стабильность слежения 
 `hit_rate_50_percent`  % времени в зоне 50px  Эффективность слежения 

Расчетные метрики в PDF
- `tracking_stability` - стабильность слежения
- `avg_tracking_duration` - время удержания цели

### 7. Переключение внимания (Attention Switching)
Суть теста Быстро переключаться между разными задачами
Специфические метрики
 Поле  Описание  Формула  Интерпретация 
----------------------------------------
 `switch_cost`  Стоимость переключения  post_time - pre_time  Когнитивная гибкость 
 `avg_pre_switch`  Время до переключения  -  Базовая скорость 
 `avg_post_switch`  Время после переключения  -  Скорость в новых условиях 

Ключевая метрика
- `switch_cost  0.1с` - отличная когнитивная гибкость
- `switch_cost 0.1-0.2с` - хорошая гибкость
- `switch_cost 0.2-0.3с` - умеренная гибкость
- `switch_cost  0.3с` - инертность мышления

### 8. Оперативная память (Working Memory)
Суть теста Удержание и обработка информации
Специфические метрики
 Поле  Описание  Интерпретация 
-------------------------------
 `memory_capacity`  Объем запоминаемых элементов  Прямая оценка памяти 
 `correct_count`  Правильно воспроизведенных  Качество запоминания 
 `total_trials`  Всего элементов  Сложность задачи 

Расчетные метрики в PDF
- `memory_efficiency` = (correct_count  total_trials) × 100%

### 9. Различение размеров (Size Discrimination)
Суть теста Точное различение близких по размеру объектов
Специфические метрики
 Поле  Описание  Интерпретация 
-------------------------------
 `avg_size_error`  Средняя ошибка (px)  Точность восприятия 
 `all_errors`  Все ошибки (JSON array)  Анализ паттернов 
 `avg_difficulty`  Средняя сложность  Прогресс обучения 

Расчетные метрики в PDF
- `size_perception_threshold` - минимальная различимая разница

### 10. Различение цветов (Color Discrimination)
Суть теста Точное различение близких цветов
Специфические метрики
 Поле  Описание  Интерпретация 
-------------------------------
 `avg_difficulty`  Средняя сложность цвета  Чувствительность к цвету 

Расчетные метрики в PDF
- `color_perception_skill` - общий навык цветоразличения

### 11. Тест Струпа (Stroop Test)
Суть теста Подавление автоматических реакций
Специфические метрики
 Поле  Описание  Формула  Интерпретация 
----------------------------------------
 `stroop_effect`  Эффект Струпа  incongruent - congruent  Стоимость интерференции 
 `congruent_rt`  Время для конгруэнтных  -  Базовая скорость чтения 
 `incongruent_rt`  Время для инконгруэнтных  -  Скорость с интерференцией 

Ключевая метрика
- `stroop_effect  0.1с` - отличный когнитивный контроль
- `stroop_effect 0.1-0.2с` - хороший контроль
- `stroop_effect 0.2-0.3с` - умеренный контроль
- `stroop_effect  0.3с` - трудности с подавлением автоматизмов

### 12. Таблицы Горбова-Шульте
Суть теста Поиск чисел с переключением внимания
Специфические метрики
 Поле  Описание  Интерпретация 
-------------------------------
 `avg_search_time`  Среднее время поиска  Скорость сканирования 
 `search_accuracy`  Точность поиска  Качество внимания 

Расчетные метрики в PDF
- `search_efficiency` = accuracy  search_time × 1000
- `memory_load` = 'high' (без подсказки) или 'low' (с подсказкой)

### 13. Комбинированные тесты A и B
Суть тестов Многозадачность и комплексные навыки
Специфические метрики
 Поле  Описание  Интерпретация 
-------------------------------
 `avg_combined_time`  Время реакции в многозадачности  Эффективность распределения внимания 
 `multi_task_efficiency`  Эффективность многозадачности  Интегральный показатель 

Ключевая метрика
- `multi_task_score` = (accuracy × 0.7) + ((100 - time×50) × 0.3)

## 🎯 КЛЮЧЕВЫЕ ВЫВОДЫ ПО МЕТРИКАМ

### 1. Латентное время (latency) - король метрик
- Показывает чистую скорость когнитивной обработки
- Менее подвержено влиянию моторных навыков, чем total_rt
- Лучший индикатор когнитивных способностей

### 2. Точность vs Скорость
- Высокая точность + низкая скорость = осторожный стиль
- Низкая точность + высокая скорость = импульсивный стиль
- Идеал баланс (высокая точность + оптимальная скорость)

### 3. Стабильность (std) - индикатор надежности
- Низкое std = стабильная работа
- Высокое std = нестабильность, возможно утомление или проблемы с концентрацией

### 4. Эффективность многозадачности
- Сравнение отдельных тестов с комбинированными
- Рост `combined_time` относительно базовых тестов = стоимость многозадачности

## ⚠️ ВАЖНЫЕ ПРЕДУПРЕЖДЕНИЯ

### 1. Контекст имеет значение
- Усталость влияет на все метрики
- Мотивация влияет на скоростьточность
- Опыт работы с компьютером влияет на моторные метрики

### 2. Не переоценивайте единичные измерения
- Однократный тест = снимок состояния
- Серия тестов = тренд, закономерность

### 3. Учитывайте индивидуальные различия
- Возрастные нормы отличаются
- Пол может влиять на некоторые метрики
- Предыдущий опыт имеет значение

### 4. Технические факторы
- Задержка монитора влияет на время реакции
- Чувствительность мыши влияет на моторное время
- Разрешение экрана влияет на пространственные метрики
